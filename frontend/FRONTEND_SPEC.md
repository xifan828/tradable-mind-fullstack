# Tradable Mind — React/JS Frontend Reconstruction Spec

This document contains everything needed to rebuild the Streamlit frontend as a React + TypeScript application. It covers all pages, components, data flows, API contracts, visual design tokens, and streaming behavior.

---

## 1. App Overview

**Tradable Mind** is an AI-powered technical analysis platform. The frontend has two views:

1. **Landing Page** — unauthenticated entry point where the user provides a Gemini API key
2. **Main App** — authenticated trading analysis interface with a chart + AI chat

**Layout:** Sidebar (left, collapsible) + main content area (right)

---

## 2. Tech Stack Recommendations

| Concern | Library |
|---|---|
| Framework | React 18 + TypeScript |
| Charts | Plotly.js (`react-plotly.js`) |
| Styling | Tailwind CSS (or CSS Modules) |
| State | Zustand or React Context |
| Streaming | `EventSource` / `ReadableStream` (SSE from backend) |
| HTTP client | `fetch` / `axios` |
| Markdown rendering | `react-markdown` + `remark-gfm` |
| Icons | Material Symbols (Google Fonts) |

## 11. Streaming Architecture

### 11.1 Talking Directly to the LangGraph Server

There is **no custom `/api/chat/stream` backend endpoint**. The agent is a LangGraph graph (`./src/agent/graph.py:graph`, see `backend/langgraph.json`) served by `langgraph dev`, which boots a server at:

```
http://127.0.0.1:2024
```

The React app talks to this server **directly** using the **LangGraph JS SDK** (`@langchain/langgraph-sdk`), the TypeScript counterpart of the `langgraph_sdk` Python client used in `backend/examples/test_api.py`. Install it in the frontend:

```bash
npm install @langchain/langgraph-sdk
```

> In production the LangGraph server is typically reverse-proxied behind the same origin as the FastAPI app (see `backend/src/agent/app.py`, which mounts the React build under `/app`), but during local development point the client straight at `http://127.0.0.1:2024`.

### 11.2 Client Setup, Threads & Runs

Mirror the Python example's flow: create a `Client`, create (or reuse) a thread, then stream a run on it.

```ts
import { Client } from "@langchain/langgraph-sdk";

const client = new Client({ apiUrl: "http://127.0.0.1:2024" });

// Create once per conversation; persist as `threadId` in app state.
// "Clear Conversation" creates a brand-new thread.
const thread = await client.threads.create();
const threadId = thread.thread_id;
```

The runtime **context** passed with each run mirrors the backend's `OrchestratorContext` dataclass (`backend/src/agent/states_and_contexts/technical_analysis.py`):

```ts
interface AgentRunContext {
  api_key: string;                  // user's Gemini API key
  asset_type: AssetType;            // "forex" | "commodity" | "crypto" | "stock"
  model_name: string;               // MODEL_MAP[agentModelLabel]
  subagent_model_name: string;      // MODEL_MAP[subagentModelLabel]
  min_research_iterations: number;  // 1–6, default 2
  max_research_iterations: number;  // 1–6, default 6
  max_concurrent_tasks: number;     // 1–4, default 4
}
```

Streaming a turn (analogous to `client.runs.stream(...)` in the Python example):

```ts
const streamResponse = client.runs.stream(threadId, "agent", {
  input: {
    messages: [{ role: "human", content: query }],
  },
  context: {
    api_key: geminiApiKey,
    asset_type: currentAssetType,
    model_name: MODEL_MAP[agentModelLabel],
    subagent_model_name: MODEL_MAP[subagentModelLabel],
    min_research_iterations: minResearchIterations,
    max_research_iterations: maxResearchIterations,
    max_concurrent_tasks: maxConcurrentTasks,
  },
  streamMode: ["messages-tuple", "updates", "custom"],
});

for await (const chunk of streamResponse) {
  // chunk.event: e.g. "messages/partial", "messages/complete", "updates", "custom", "metadata", "error"
  // chunk.data:  payload shape depends on chunk.event
  handleStreamChunk(chunk);
}
```

`"agent"` is the assistant/graph name declared in `backend/langgraph.json` (`graphs.agent`). The `threadId` is reused for every message in the same conversation (thread continuity, §15.5) and only replaced when the user clicks "Clear Conversation".

### 11.3 Mapping Raw Chunks to UI Events

The chunks yielded by `client.runs.stream` are the LangGraph server's native stream events — there is no backend-side translation layer, so the frontend is responsible for interpreting them and deriving the same conceptual events the chat UI renders. Internally, normalize each raw chunk into a `StreamEvent`:

```ts
interface StreamEvent {
  event_type: "thinking" | "tool_call" | "task" | "tasks_collected" | "tool_result" | "todos" | "text" | "done" | "error";
  content: any;
  tool_name?: string;
  tool_call_id?: string;
}
```

Derivation rules from raw chunks:

| Raw `chunk.event` | `chunk.data` contains | Derived `StreamEvent` |
|---|---|---|
| `metadata` | run/thread identifiers | (ignored by UI) |
| `messages/partial`, `messages/complete` | streamed `AIMessage` chunks — `content` (text or reasoning blocks) and `tool_calls` | `thinking` (reasoning/thought content), `text` (final assistant prose), `tool_call` (when `tool_calls` is populated, e.g. `write_todos`, `task`) |
| `updates` | per-node state diffs (e.g. orchestrator emitting `todos`, subagents returning `ToolMessage` results) | `todos`, `tool_result` (matched to the originating `tool_call_id`) |
| `custom` | any custom events the graph dispatches | `task` / `tasks_collected` when the orchestrator delegates parallel chart/quant work |
| `error` | error payload (string or object) | `error` |
| stream end (generator exhausted) | — | `done` |

Once normalized into `StreamEvent`s, the rest of the chat pipeline behaves exactly as described below — this keeps the UI logic decoupled from the transport details of the LangGraph SDK.

### 11.4 Event Handling Logic

```
thinking        → finalize pending iteration → show 💭 Thinking block
tool_call       → finalize pending iteration → show tool call block
task            → start new iteration group if none open → add task to group
tasks_collected → show the iteration block immediately (before results)
tool_result     → match by tool_call_id → update task result in-place + update header
todos           → update current investigation plan display
text            → finalize pending iteration → show final markdown response
done            → finalize any open iteration → clear status indicator
error           → show error message
```

### 11.5 Iteration State During Streaming

```ts
interface IterationState {
  iterationNum: number;
  tasks: TaskState[];
  displayed: boolean;  // true once rendered to DOM
}

interface TaskState {
  taskArgs: {
    task_type: string;
    task_description: string;
    chart_analysis_input?: Record<string, any>;
  };
  result: string | null;
  toolCallId: string;
}
```

## 15. Key Behavioral Rules

1. **API key gate**: Show landing page if no key in state. Store key in localStorage. On "Get Started", navigate to `/app`.

2. **Chart first**: The chat section only appears after the chart is loaded (`chartLoaded === true`).

3. **Streaming lock**: When `isStreaming`, disable ALL sidebar controls and the chat input. Show warning banner in sidebar.

4. **Pending actions**: If user clicks "Load Chart" or "Clear Conversation" while streaming, queue the action and execute after the stream ends.

5. **Thread continuity**: `threadId` persists across messages within a conversation. Only reset on "Clear Conversation".

6. **Real-time task updates**: Investigation Round results update IN-PLACE as they arrive — the round block shows ⏳ for each pending task and updates to show result + ✅ when `tool_result` arrives, matched by `tool_call_id`.

7. **Daily change vs interval change**: The price change shown in the header uses separately fetched daily OHLC data, NOT the most recent interval bar's change.

8. **X-axis as categorical**: Plotly x-axis type must be `"category"` to avoid visual gaps on weekends/holidays.

9. **Free tier API key error**: If backend returns error containing "free_tier", show a specific user-friendly error message pointing to the rate limits page.

10. **Theme persistence**: Store theme preference in localStorage.

## 16. Empty / Loading States

| Scenario | UI |
|---|---|
| No chart loaded | Centered info block: "To begin: Enter an asset symbol, Choose your indicators, Press Load Chart" |
| Chart loading | Spinner/skeleton with "Loading data for {SYMBOL}..." |
| Chart load error | Error alert with message |
| No API key in chat | Warning: "Enter your Gemini API key in the sidebar to enable AI analysis chat." |
| Agent analyzing | Info banner: "Agent is analyzing..." with a brain/psychology icon |
| Stream error | Error message in chat |
